# Quality Rubric — 20-Point Scale

> Универсальный рубрикатор для оценки артефактов всех агентов. Используется `quality-controller`. Применяется одинаково к постам, outbound сообщениям, SEO секциям, гипотезам, ICP-валидациям и пр.

## Структура — 5 категорий, 20 баллов

| Категория | Макс | Что оцениваем |
|-----------|------|---------------|
| **A. Adherence** | 5 | Выполнил ли всё из промпта, прочитал ли нужные файлы |
| **B. Factual accuracy** | 5 | Числа из proof-points.md, нет halluc, верный продукт |
| **C. Brand & tone** | 3 | Tone of voice, no-go list, no AI signatures |
| **D. Format & structure** | 3 | Frontmatter, путь файла, структура по шаблону |
| **E. Output quality** | 4 | Готовность к использованию, уникальная ценность |
| **Total** | **20** | |

## A. Adherence to instructions (0-5)

| Score | Описание |
|-------|----------|
| 5 | Выполнено всё из промпта. Все входные файлы прочитаны. Все требования к выводу учтены. |
| 4 | Выполнено всё, но один шаг сделан поверхностно (например, прочитал proof-points.md но не использовал релевантные числа). |
| 3 | Пропущен 1 важный шаг ИЛИ не прочитан 1 ключевой файл (e.g., не открыл messaging.md перед написанием). |
| 2 | Пропущено 2+ шага из алгоритма промпта. |
| 1 | Явно срезал углы — выполнил только базу. |
| 0 | Не выполнил основную задачу, либо вышел за scope. |

## B. Factual accuracy & anti-hallucination (0-5)

| Score | Описание |
|-------|----------|
| 5 | Все числа из `proof-points.md`. Продукт верно идентифицирован. Кейсы релевантны (FX-кейсы для FX, MT для MT). |
| 4 | Все числа верны, минорная ошибка в позиционировании или формулировке (e.g., «96% accuracy» вместо «96-97%»). |
| 3 | 1 число не из proof-points (но не критичное), либо 1 неверная атрибуция кейса. |
| 2 | 2+ непроверенных числа, либо явная подмена (Safariland в FX-кампании). |
| 1 | Значительная фактическая ошибка влияющая на доверие. |
| 0 | Полностью выдуманные данные / кейсы / клиенты. |

**Hard fail (auto 0 в этой категории):**
- Выдуманные клиенты которых нет в `case-studies/`
- Выдуманные числа которых нет в `proof-points.md`
- Лидирование с «most accurate scanning» (anti-positioning violation)

## C. Brand & tone compliance (0-3)

| Score | Описание |
|-------|----------|
| 3 | Tone соответствует CLAUDE.md секция 6. Нет banned words. Нет AI signatures (em-dash в риторических конструкциях, «not just X, it's Y»). |
| 2 | 1-2 minor lapse (одно banned word или один em-dash). |
| 1 | Несколько lapses или один заметный (3+ banned words либо тон явно не Вадима). |
| 0 | Грубое нарушение тона / clickbait / много AI signatures. |

**Banned words checklist** (из `messaging.md`): leverage, utilize, harness, robust, seamless, comprehensive, delve, navigate (метаф.), tapestry, realm, unlock, unleash, game-changing, cutting-edge.

## D. Format & structure (0-3)

| Score | Описание |
|-------|----------|
| 3 | Frontmatter полный (`product:`, `status:`, etc.). Файл в правильном пути. Структура полностью по шаблону из промпта. Char counts соблюдены (для сообщений / meta). |
| 2 | Minor отклонение (один пропущенный frontmatter field, либо char count чуть превышен). |
| 1 | Структура частично нарушена (отсутствуют 2+ обязательных поля, либо неверный путь). |
| 0 | Формат явно нарушен — нет frontmatter, или файл не там где должен быть. |

## E. Output quality (0-4)

| Score | Описание |
|-------|----------|
| 4 | Готово к использованию as-is. Есть уникальная ценность / угол. Текст звучит как сделанный профессионалом, не AI. |
| 3 | С минимальными правками годится для публикации (5-10 минут редактуры Вадима). |
| 2 | Нужны умеренные правки (30+ минут). Идея есть, но исполнение слабое. |
| 1 | Значительные переработки. Структура есть, но содержимое generic. |
| 0 | Непригодно. Нужна полная регенерация с другим подходом. |

## Итоговая оценка

| Total | Статус | Действие |
|-------|--------|----------|
| 18-20 | ⭐ Excellent | Утверждать без правок, сохранить как образец |
| 15-17 | ✅ Good | Утверждать с минорными правками |
| 12-14 | ⚠️ Marginal | Запросить переработку |
| 0-11 | ❌ Failed | Регенерировать с другим подходом, передать improver-у |

## Per-track overrides

Некоторые треки имеют специфичные критерии внутри категорий:

### Outbound message
- **B (Factual):** обязателен compliance mention если ICP healthcare/insurance
- **E (Quality):** проверка персонализации — каждое сообщение должно быть уникально на 60%+ от других

### SEO section / final
- **B (Factual):** все источники из outline должны быть процитированы только если реально открыты (WebFetch)
- **E (Quality):** AI detector score (если доступен инструмент) — для финального текста ≤ 25%

### SEO meta
- **D (Format):** title 50-60 chars, description 140-160 chars — жёсткие лимиты, выход за них = 0 в этой категории
- **B (Factual):** primary keyword в первой половине title — обязательно

### Social post
- **C (Brand):** сравнение с 3+ постами из past-posts/{profile}/ на стилевую близость
- **E (Quality):** есть ли hook в первых 2 строках (visible до «see more»)

### Visual brief
- **D (Format):** все 5 секций обязательны (Format / Concept / Composition / Copy / Tech specs)
- **E (Quality):** дизайнер должен суметь сделать визуал без дополнительных вопросов
